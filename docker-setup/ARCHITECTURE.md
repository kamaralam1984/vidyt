## 🚀 30-Second Start

```bash

# Follow prompts, then:
```

## 📁 What You Get

### Services (All Production-Ready)
1. **Frontend** - Next.js SSR application (port 3000)
2. **Backend** - Express.js REST API (port 5000)
3. **AI Service** - Python Flask ML service (port 5001)
4. **Workers** - 2x BullMQ queue processors (ports 5002-5003)
5. **Database** - MongoDB document store
6. **Cache/Queue** - Redis (cache + job queue)
7. **Proxy** - NGINX reverse proxy with SSL, rate limiting, compression

### Infrastructure
- ✅ Multi-stage Docker builds (optimized, small images)
- ✅ Health checks on all services
- ✅ Proper restart policies
- ✅ Volume management (persistent data)
- ✅ Network isolation (bridge network)
- ✅ Security hardened (non-root users, rate limits, security headers)
- ✅ Logging configured (json-file driver)
- ✅ Scalable worker architecture

## 🚀 Production Deployment

### Quick Path
```bash
# 1. SSH into VPS
ssh root@your-vps-ip

# 2. Clone & setup
git clone <repo> && cd docker-setup
./setup.sh

# 3. Update .env
nano .env

# 4. Get SSL certs (Let's Encrypt)
certbot certonly --standalone -d yourdomain.com
cp /etc/letsencrypt/live/yourdomain.com/* nginx/ssl/

# 5. Start
docker-compose up -d

# 6. Setup DNS
# Point A record to VPS IP
```

## 🛠️ Troubleshooting Quick Guide

### Services won't start?
```bash
docker-compose logs --tail=50
docker-compose build --no-cache
```

### Port conflict?
```bash
lsof -i :5000
kill -9 <PID>
# Or edit ports in docker-compose.yml
```

### Database connection failed?
```bash
docker-compose exec mongodb mongosh -u root
# Test from another terminal
```

### Out of disk space?
```bash
docker system prune -a    # Remove all unused
docker volume prune       # Remove unused volumes
```

See `DEPLOYMENT_GUIDE.md` **Troubleshooting** section for more.
